# Findings

## Market and trend signals
- Agent tooling is growing rapidly, especially projects focused on autonomy and developer workflows.
- A common pain point is poor debuggability for agent runs.
- Local-first and protocol-friendly tooling (for MCP ecosystems) is attractive and shareable.

## Product positioning
- Build an observability layer, not another agent runtime.
- Make replay and failure diagnosis the core "wow moment."
- Keep setup friction near zero to maximize adoption.
